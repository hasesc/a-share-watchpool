# v0.9.2 Release Notes Draft

Status: Draft  
Target: offline examples, schema documentation, and validation

## Summary

v0.9.2 focuses on making A股观察池 · A-Share Watchpool understandable and testable without live market data. The release should give reviewers and contributors enough fictional fixtures to inspect the data shape while preserving the project boundary.

This release does not add live brokerage access, real trading commands, private data ingestion, or investment advice.

## Included Changes

- Add fictional or anonymized example fixtures under `examples/`.
- Document example schemas for:
  - Data health results.
  - Watchlist entries.
  - Offline report summaries.
- Add a standard-library-only validator for offline examples.
- Add tests that reject broker-like, account-like, or real-order-like fields.
- Run offline example validation in CI before pytest.

## Fixture Rules

All example fixtures must be:

- Fictional or anonymized.
- Independent of live market endpoints.
- Free of real account data, cookies, API keys, tokens, and personal information.
- Free of broker API fields and real buy/sell order fields.
- Clear that they are not investment advice.

## Out Of Scope

This release intentionally does not add:

- Real trading records.
- Real account data.
- Broker API samples.
- Automatic order execution samples.
- Return promises or recommendation language.
